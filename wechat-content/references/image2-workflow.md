# Image 2 配图工作流

## 默认行为

- 新闻采用“七天七色”：星期一至星期日各有独立的无字封面与总览图，离线电脑也能生成完整审核包。
- 第八套中性素材只作为配置或日期异常时的默认兜底，不参与正常星期轮换。
- 准确中文标题、栏目名和序号由本地 Pillow 叠加，不交给图像模型生成。
- GitHub 热门沿用自己的开源视觉，不读取新闻星期素材。
- 输出中记录 `image_mode`：当期有效输入为 `live_image2`，内置星期图为 `weekday_fallback`。

## 当期动态图片

1. 根据当天入选新闻生成两张无文字横图：`cover.png` 与 `overview.png`。
2. 图中不得出现标题、日期、Logo、水印、虚构界面或需要程序对齐的空白标签框。
3. 将两张图放入同一目录，通过 `--image-input-dir <目录>` 传入。
4. 渲染器会校验格式、尺寸和文件体积；两张都有效才标记 `live_image2`。
5. 任一图片缺失或无效时，整组切换到对应星期的 `weekday_fallback`，并在运行报告记录原因，不混用半组图片。

## 更新底图

1. 只生成无字主视觉，不包含日期、Logo、水印、产品界面、品牌标识或等待程序填充的空白标签框。
2. 新闻图表达多条公共信息之间的联系，GitHub 图表达探索开源项目，不可混用。
3. 人工检查主体、裁切安全区和不当符号后，才可替换 `assets/` 中的底图。
4. 重新运行 Skill，由本地程序制作 900×383 横版、383×383 方形及左长右方合并封面。
5. 检查每张图片小于 2 MB，并在公众号手机预览中复核。

星期素材缺失时，渲染器使用中性默认兜底；不能因此跳过正文，也不得把兜底素材声称为本期 Image 2 动态生成结果。
