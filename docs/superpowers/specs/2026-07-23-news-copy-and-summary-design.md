# 新闻复制门槛与速览排版修复设计

## 目标

修复 `daily-news` 内容包存在完整正文但因部分核验状态无法复制的问题，同时修复“30 秒速览”字符串被逐字拆分和关键词胶囊样式过碎的问题。

## 状态与复制规则

- `verified`：允许复制。
- `partial`：正文五项发布字段完整且来源链接存在时允许复制，复制区外显示“部分信息需发布前复核”。
- `unverified`：禁止复制。
- 任一新闻缺少 `what_happened`、`why_it_matters`、`reader_action`、`editor_note` 或 `keywords`：禁止复制。
- 整体 `status` 继续忠实反映研究质量；复制资格单独计算，不通过篡改 `needs_review` 绕过审核。

研究层在事件聚类时必须保留已经完成的 `partial` 和 `verified` 状态及其证据，不得把 `partial` 降为 `unverified`。

## 速览标准化

`editorial.overview` 支持字符串或字符串列表。字符串按中文分号、句号和换行拆分，去除空项并限制为 3—6 条；列表保持逐条输出。禁止直接遍历字符串字符。

## 关键词样式

恢复连续信息条：浅色圆角背景、加粗“关键词：”、关键词使用 `｜` 分隔并自然换行。取消每个词的独立胶囊，颜色继续跟随每日主题。

## 验证

- 完整 `verified + partial` 内容包允许复制并显示复核提醒。
- 含 `unverified` 或缺字段的内容包禁止复制。
- 字符串速览不会逐字拆分。
- 关键词以单条信息栏呈现，无 `keyword-chip`。
- GitHub 栏目行为不变。
