# 新闻提示标题按内容自动选择设计

## 目标

移除每日新闻正文中固定出现的“小清提醒”，根据每条新闻的内容自动选择更自然的提示标题。只改变提示标题，不改写研究内容包中的 `editor_note`，不影响 GitHub 热门栏目。

## 选择规则

渲染器根据新闻条目的 `category`、`keywords`、标题和说明字段进行确定性匹配，按以下优先级选择：

1. 数据、争议、传闻、辟谣或信息尚不完整：`边界说明`
2. 天气、灾害、交通、公共安全：`实用提醒`
3. 仍在发展、等待后续通报或政策尚待落地：`接下来关注`
4. 政策、民生、教育、医疗、消费：`与你有关`
5. 其他新闻或无法识别：`值得留意`

高风险的“边界说明”优先于其他类别，避免争议信息被包装成普通生活建议。同一内容包重复渲染必须得到相同标题，不调用随机数或联网模型。

## 实现边界

- 新增一个独立的提示标题选择函数，输入单条新闻字典，输出上述五个标题之一。
- Markdown 和微信 HTML 继续使用现有 `editor-note` 信息块，仅把固定标题替换为选择结果。
- 输入缺少 `editor_note` 时，仍沿用现有 `needs_review` 质量降级，不用自动标题掩盖内容缺失。
- 更新每日新闻规范，删除“小清提醒”称呼并说明选择规则。
- GitHub 热门构建路径不得调用该函数。

## 验证

- 单元测试覆盖五种标题及默认行为。
- 回归测试确认旧称呼“小清提醒”不再出现在新闻 Markdown 和 HTML 中。
- 回归测试确认 `editor_note` 原文保持不变。
- 运行 `wechat-content` 全套测试，并使用真实新闻内容包重新生成预览。
