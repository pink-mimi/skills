# 新闻读者正文净化设计

## 目标

让 `wechat-content` 生成的每日新闻复制正文只包含适合读者阅读的内容。内部审核状态、内容包术语和待补充占位语不得进入复制区域；新闻提示标题、结尾说明、后续观察和来源区域根据当日内容稳定生成。GitHub 热门栏目不受新闻规则影响。

## 读者正文与审核信息分离

- 统计时段只显示北京时间范围，删除“内容包核验时刻”等内部说明。
- `needs_review`、缺失字段、图片降级和“尚未发布”等状态仅显示在复制区域外的审核提示中。
- 新闻缺少 `what_happened`、`why_it_matters`、`reader_action`、`editor_note` 或 `keywords` 时，继续把输入状态降级为 `needs_review`，同时禁用复制按钮。
- 读者正文不得出现“待人工补充”“内容包未提供”“发布前请补充”“人工审核包”或“尚未发布”。
- 完整内容包保持现有一键复制体验。

## 新闻提示标题

标题选择采用确定性分层规则：

1. 主要依据 `category`、`keywords` 和 `title`。
2. `summary` 仅作为辅助信号。
3. `what_happened`、`why_it_matters`、`reader_action` 和 `editor_note` 不参与标题分类，避免行动建议中的“安全”“通报”等普通词造成误判。
4. 优先级维持“边界说明 → 实用提醒 → 接下来关注 → 与你有关 → 值得留意”。
5. 同一内容包重复生成必须得到相同标题。

## 动态结尾说明

结尾说明从当天全部入选新闻中确定需要的提示，按固定顺序去重合并：

- 天气、灾害、交通和公共安全：提醒关注属地权威预警与通报。
- 财经和市场：提醒数据可能随交易、统计周期和口径变化。
- 政策：提醒以主管部门正式文件和实际执行安排为准。
- 争议、传闻、辟谣和尚未证实信息：提醒关注后续权威核实。
- 没有特殊类别：使用“本文依据公开资料整理，相关信息请以原始来源最新内容为准。”

说明中不得出现审核状态和发布流程。多种提示合并为一个简洁段落，不逐条重复免责声明。

## 后续观察与来源

- 栏目名从“信息来源与动态说明”简化为“参考来源”。
- 每条来源保留可点击标题和下一行可见原始网址，以兼顾微信外链限制和人工追溯。
- “今天值得关注”优先使用内容包 `editorial.follow_up`。
- 将规范化后的观察点与新闻标题比较；完全相同或高度重复的内容移除，同时对观察点自身去重。
- 没有有效观察点时不输出“今天值得关注”栏目。

## HTML 复制行为

- 完整内容包：复制按钮可用，仅复制 `#wechat-content`。
- `needs_review`：复制按钮设为禁用，并在复制区域外显示明确的人工补全提示。
- 工具栏、封面预览、审核提示、运行状态和备用标题均不进入复制内容。
- 来源链接继续以标准 `<a>` 输出，并保留可见 URL。

## 验证

- 测试统计时段和结尾不含内部术语。
- 测试天气、市场、政策、争议信息和中性兜底说明。
- 测试提示标题不会被 `reader_action` 或 `editor_note` 中的偶然关键词误导。
- 测试不完整内容包禁用复制按钮，且复制区域无内部占位语。
- 测试后续观察去重、空栏目省略和来源两行结构。
- 测试 GitHub 热门输出不应用新闻净化规则。
- 使用真实新闻内容包重新生成并检查微信 HTML。
