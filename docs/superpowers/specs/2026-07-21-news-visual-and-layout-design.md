# 新闻七天视觉与公众号排版设计

## 目标

升级 `wechat-content` 的 `daily-news` 栏目，使新闻每天保持统一品牌风格但不重复同一画面，同时恢复 2026 年 7 月 20 日版本更清晰、连续的公众号阅读体验。

本设计只调整新闻栏目。`github-hot` 继续使用独立的文章结构、配图语义、主题和项目卡片，不得套用新闻视觉。

## 栏目路由

`wechat-content` 必须先读取 `content_type`，再选择栏目模板：

- `daily-news`：新闻结构、新闻配图与新闻排版。
- `github-hot`：项目结构、开源探索配图与项目信息排版。
- 新增内容类型时必须新增栏目规范、渲染器、图片语义、fixture 和回归测试，不能只替换标题。

视觉主题只控制栏目内部的颜色和装饰，不得覆盖栏目结构。

## 新闻图片策略

采用“动态生成优先，七天素材轮换降级”的混合模式。

### 统一视觉语言

- 冰白、浅天蓝、深蓝路线与少量暖橙坐标。
- 立体微缩、纸雕质感、地图与连续路线。
- 不生成真实新闻现场、产品界面、品牌 Logo 或人物身份。
- Image 2 只生成无字画面；中文标题由本地程序叠加。
- 禁止模型生成日期、文字、数字、标签、卡片和空白标签框。

### 实时生成

当运行环境具有 Image 2 能力时，根据当期入选新闻的标题、类别和关键词生成：

1. 一张无字封面主视觉。
2. 一张无字正文新闻脉络图。

两张图片共享当期主题和视觉语言，但正文图不是封面图的简单裁切。生成后检查文字、水印、品牌标识、空白标签框、主体安全区、尺寸和体积；任一检查失败则使用备用素材。

### 七天备用图库

首版内置 7 套新闻备用视觉，每套包含无字封面底图、无字正文脉络图、安全区和版本信息：

1. 周一：城市与公共服务。
2. 周二：科技与产业。
3. 周三：教育与社会。
4. 周四：经济与生活。
5. 周五：文化与趋势。
6. 周六：公共安全与出行。
7. 周日：一周观察与综合脉络。

七套素材使用同一主风格，只改变场景构成、路线方向和视觉重心。备用选择以北京时间星期为准；同一内容包重复执行必须选择相同素材。

### 输出记录

`render-manifest.json` 与 `运行报告.md` 记录：

- `image_mode`：`live_image2` 或 `weekday_fallback`。
- `visual_variant`：星期与素材编号。
- `asset_version`：素材版本。
- `fallback_reason`：实时生成不可用或检查失败的原因；实时生成成功时为空。

Skill 本地脚本不能假装已经调用 Image 2。只有运行代理实际生成并提供合格图片时，才能记录 `live_image2`。

## 封面输出

- 合并封面固定为 `1283×383`，左侧 `900×383`，右侧 `383×383`。
- 同时导出横版和方形独立文件。
- 封面不出现日期。
- 标题根据安全区按像素宽度换行，不覆盖核心场景。
- 方形封面使用独立安全区配置，不直接采用不可控的中心裁切。

## 新闻正文排版

正文采用“旧版阅读体验＋新版完整内容结构”。

固定顺序：统计时段、30 秒速览、新闻脉络图、逐条解释、今天值得关注、来源说明、结尾互动。

### 视觉层级

- 页面主色恢复为深蓝，正文保持白底连续阅读。
- 统计时段使用淡蓝底和一条完整的深蓝左边线，不得出现脱离内容框的竖线。
- “30 秒速览”使用紧凑的项目符号、缩进和行距。
- 新闻主标题使用深蓝粗体，与正文保持明确字号差。
- 关键词只使用一条完整淡蓝信息栏。
- “发生了什么”“为什么重要”“普通人需要注意什么”使用纯文字粗体小标题，不分别套浅色卡片。
- “小清提醒”是每条新闻唯一的重点提示块。
- 减少大段空白、浅青色文字和重复卡片，适配手机连续阅读。
- 保留当前结尾图和有意义的互动结尾。

### 来源链接

- HTML 保留标准 `<a href>` 外部链接。
- 同时显示原始网址，作为未认证公众号清除正文外链时的追溯降级。
- 不得声称 Skill 能绕过微信外链权限。
- 人工发布时可选择一条主要来源设置为“阅读原文”。

## 错误处理

- Image 2 不可用：使用当日星期备用图，审核包继续生成。
- 实时图不合格：记录检查原因并使用备用图。
- 星期素材缺失：使用综合新闻备用图并标记 `fallback_reason`。
- 全部备用图缺失：使用程序化新闻模板，状态标记 `needs_review`，不得使用空白占位图。
- 内容包缺少发布级解释字段：继续标记 `needs_review`，视觉生成不得掩盖内容不足。

## 验证标准

- 连续 7 个北京时间日期分别选择 7 套素材，第 8 天稳定回到第一套。
- 同一内容包重复运行，图片选择与 manifest 保持一致。
- 新闻与 GitHub fixture 生成的模板、图片文件名和正文结构不同。
- 统计时段左边线完整，正文小标题不使用独立卡片。
- HTML 复制区域包含内联样式、嵌入图片、标准链接和可见网址。
- 合并、横版、方形和正文图尺寸正确，单张输出小于 2 MB。
- Image 2 成功、不可用、检查失败和素材缺失路径均有测试。
- 只生成审核包，不上传或发布公众号。

## 实施边界

- 本次修改 `wechat-content` 的新闻图片与新闻排版，不重写新闻采集逻辑。
- 不改变 `github-hot` 已有栏目结构和视觉。
- 不在 Skill 内保存密钥、Cookie 或公众号后台数据。
- 定时任务负责每天调用，Skill 只按传入内容包和北京时间确定视觉变体。
