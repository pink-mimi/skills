# 新闻采集平台与覆盖说明

## 来源平台

| 类别 | 默认日报 | 平台与来源 | 采集方式 | 定位与限制 |
|---|---|---|---|---|
| 时政 | 是 | 人民网·时政 | RSS | 重要政策仍需回到政府部门原文核验 |
| 财经 | 是 | 人民网·财经 | RSS | 市场数字和公司数据发布前再次复核 |
| 社会 | 是 | 人民网·社会、中国新闻网 | RSS | 公共安全优先寻找官方机构原文 |
| 国际 | 是 | 人民网·国际 | RSS | 属于中文媒体国际报道，不等于海外原始信源 |
| 科技 | 是 | 人民网·科技、IT之家 | RSS | 产品信息需要核对厂商公告 |
| 体育 | 否 | 新浪体育 | 网页候选发现 | 页面结构变化可能导致解析失败 |
| 娱乐 | 否 | 新浪娱乐 | 网页候选发现 | 只在明确查询时启用，不进入默认日报 |
| AI 技术 | 否 | MIT Technology Review、Google AI Blog、Google DeepMind | RSS | 英文来源，需要翻译并核对原文发布日期 |
| AI 社区 | 否 | Hacker News、Product Hunt | RSS | 社区热度不等于事实权威，只用于发现线索 |

## 默认内容

“昨日新闻”默认覆盖时政、财经、社会、国际和科技。体育、娱乐、AI 技术与 AI 社区不会自动混入日报，只有用户明确指定分类时才采集。

## 百分比指标

每次运行的 `source-report.md` 会记录：

- 配置来源数量；
- 成功和失败来源数量；
- 本次采集成功率；
- 各平台候选数量；
- 各类别候选数量与分布；
- 最终入选数量。

采集成功率只表示请求成功的来源占配置来源的比例，不能保证 100% 的新闻覆盖，也不能保证事实准确率达到 100%。RSS 可能延迟、漏发或改版，发布前仍需人工打开原文核验。

## 许可证说明

本 Skill 独立实现采集、解析、查询、过滤和报告能力，不在运行时依赖第三方 `$news` Skill，也不复制其未声明许可证的源代码。
