# 每日新闻栏目模板

- 栏目标识使用“昨日坐标”，不得使用“项目”命名图片。
- 开头从事件之间的联系、反差或对普通生活的影响切入，不描述采集过程。
- 固定顺序为：统计时段、30 秒速览、新闻一日脉络图、逐条解释、可选的“今天值得关注”、参考来源和动态结尾说明。
- 每条新闻呈现关键词、发生了什么、为什么重要和普通人需要注意什么。可选的 `reader_tip` 使用独立浅色提示卡并随正文复制；缺失时省略，不生成占位语。关键词使用单条浅色关键词信息条，加粗“关键词：”并以 `｜` 分隔，不使用独立胶囊。
- `editor_note`、`verification_notes` 和 `verification_status` 属于内部审核信息，原样保留在复制区域外的运营者审核区，不得包装成“与你有关”“值得留意”等读者提示。
- 标题选择是确定性的，不使用随机数或联网模型；同一内容重复渲染必须得到相同标题。
- 标题和封面预览放在复制区域之外；公众号正文复制区从统计时段开始。
- 公众号标题优先使用 `editorial.article_title`；`editorial.title` 含“内容包”“新闻包”“审核包”或“工作台”时视为内部名称，不得直接展示给读者。缺少有效文章标题时，使用统计起始日期生成中性“国内要闻”标题。
- 内部审核状态、内容包术语、待补充占位语和“尚未发布”等信息不得进入复制正文。读者正文必填字段完整时允许复制；存在 `partial` 或 `unverified` 时，在复制区域之外显示发布前核验提示，并将 `publish_ready` 保持为 `false`。只有读者正文必填字段缺失时才禁用复制。
- 结尾说明根据当天新闻动态组合：天气灾害、交通和公共安全提示权威预警，财经市场提示统计口径，政策提示正式文件，争议和未证实信息提示后续核实；没有特殊类别时使用中性来源说明。
- “今天值得关注”对观察点自身去重，并移除与新闻标题相同、互相包含或高度重复的内容；没有有效观察点时省略整个栏目。
- 来源栏目名使用“参考来源”，每项同时保留可点击的来源标题和下一行可见原始网址。
- 正文默认只使用一张“一日脉络”总览图，避免多张结构相同的卡片打断阅读。
- 封面和脉络图优先使用 Skill 内置的无字 Image 2 底图，准确中文由本地程序叠加；底图缺失时才使用程序模板。
- 正常运行按北京时间星期执行“七天七色”，每日主题不同、同一日期稳定；中性主题仅作默认兜底。
- 如果为当期新闻生成了新的无字 `cover.png` 与 `overview.png`，必须通过 `--image-input-dir` 成对传入；成功记为 `live_image2`，否则记为 `weekday_fallback`。
- 新闻星期主题不向 `github-hot` 或后续栏目泄漏；每种素材类型必须拥有自己的文章结构和图片语义。
- 不虚构影响；输入没有提供的判断不补写。
- 结尾从热度转向长期影响，并邀请读者选择希望继续追踪的线索。
- 正文图使用新闻节点、轨迹和观察视角，不生成虚构新闻现场或人物照片。Image 2 不可用时也必须生成有信息层级的脉络图，不得使用空白几何占位图。
- Image 2 底图不得预留空白标签框，程序也不得按固定坐标叠加动态类别文字；具体新闻信息只在正文中表达，避免条目数量变化后错位。
- 来源同时保留标准 `<a>` 超链接和可见原始网址。未认证公众号可能清除正文外链，网址文字作为人工复制与追溯的降级方案；不得声称 Skill 能绕过微信权限。
