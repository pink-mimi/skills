# 来源与风险核验

## 原始来源优先级

1. 仓库主页与 README：用途、平台、安装方式和适用人群。
2. LICENSE 文件及 GitHub 许可证标识：授权名称与适用范围。
3. Release 与 Commit：版本、发布日期和近期维护。
4. Issues、安全政策与官方文档：已知限制、漏洞、兼容性和数据处理。
5. Trending、Explore、官方博客和社区讨论：只作为发现或关注度线索。

每个入选项目至少保留仓库 URL、许可证证据、维护证据、核验时刻和风险说明。

## 风险检查

- 安装命令是否直接执行远程脚本。
- 是否需要账号、API Key、付费模型、云服务或特殊硬件。
- 是否上传文件、代码、笔记、密钥或个人信息。
- 是否涉及身份认证、生产数据库、金融操作、医疗或公共安全。
- 自动生成的代码、文档、公式和界面是否需要人工复核。
- “开源”不等于免费商用；以实际许可证和依赖许可证为准。

遇到限流、来源冲突或动态数据无法复核时，保留冲突与错误，不取平均值，也不写成确定事实。
