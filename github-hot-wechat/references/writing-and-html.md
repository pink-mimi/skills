# “未完地图”写作与微信 HTML

- 面向普通读者，语言通俗、克制、略带探索感；地图、坐标和路标比喻只作轻量点缀。
- 开头 100—180 字，从本周项目之间的变化、反差或共同问题切入，不讲机械搜集过程，不编造作者体验。
- 每个项目包含：一句话说明、解决的问题、2—3 个亮点、适合谁、使用门槛、维护与许可证、风险限制和官方地址。
- 结尾给出一层值得继续思考的观察，并邀请读者选择项目或推荐下期线索。
- 输出 5 个准确标题和不超过 120 字摘要。
- 避免“神器”“炸裂”“突然爆火”“闭眼入”等夸张表达。

微信 HTML 必须使用内联样式，并满足：

- 顶部提供“复制公众号正文”按钮。
- `#wechat-content` 只包裹正文；工具栏、合并封面预览和内部说明在复制范围之外。
- 优先复制富文本，同时提供纯文本和 Selection 回退。
- 图片宽度为 100%，长链接可换行，正文最大宽度 720px，无横向溢出。
- 标题与封面由人工在公众号后台分别填写；最终必须进行手机预览。
